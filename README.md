# Habit-Challenge-Fairness-System-Final-Prototype
This html is part of a Master's HCI project exploring a new validation system for competitive fitness challenges. The system combines three layers: an AI pre-check, anonymous peer voting, and a challenge leader tie-break — all working together to decide whether a submitted activity is valid. 

## 1 — Goal & Overview

This prototype explores a **hybrid validation system** for GymRats check-ins. Instead of a single validator, it chains three layers: an AI pre-check, anonymous peer voting, and a leader tie-break. Use the **role switcher** above the phone frame to experience the same check-in from different perspectives.

## 3 — Disabled in This Prototype

The following features are mocked or non-functional on purpose:

- **Notifications** — bell icon is decorative only
- **User profile** — no real account management
- **Leaderboard / ranking** — scores are static
- **In-app chat** — comment input is non-functional
- **Challenge discovery / join flow** — only one challenge shown
- **Device sync** — fitness device data is hardcoded
- **Real image upload** — images are pre-loaded assets
